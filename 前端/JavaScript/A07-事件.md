**什么是事件**
- 在编程时系统内发生的**动作**
- 如单击按钮
**什么是事件监听**
- 程序检测是否有事件发生
- 事件发生, 则调用函数做出反应


# 事件监听

- 语法
	- 元素对象.addEventListener('事件类型', 要执行的元素)
- 事件监听也成为**注册事件**

**三要素**
- 事件源 -> 获取dom元素
- 事件类型 -> 用什么方式触发
- 事件调用的函数 -> 做什么事
	- 绑定后事件触发时执行

##### 事件监听版本
- DOM L0 : 事件源.on事件名 = function(){}
- DOM L2 : 事件源.addEventListener('事件名', 函数)
- 区别: on方式会被覆盖, addEventListener方式可以绑定多次, 不会覆盖, 而且拥有事件更多特性

**历史**
- DOM L0 : DOM第一版本
- DOM L1 : 于1998年10月成为推荐版本
- DOM L2 : 使用addEventListener注册事件
- DOM L3 : 在DOM L2的基础上重新定义了这些事件

# 事件类型

##### 鼠标事件
- click 点击
- mouseenter : 鼠标进入
- mouseleave : 鼠标离开

##### 焦点事件
- focus : 获得焦点
- blur : 失去焦点

##### 键盘事件
- keydown : 键盘按下
- keyup : 键盘抬起

##### 文本事件
- input : 用户输入


# 事件对象

### 获取事件对象
- 什么是事件对象
	- 一个对象， 其中存储着事件触发时的**相关信息**
	- 例如，鼠标点击时的位置、键盘按下了哪个键等
- 场景
	- 判断用户按下了哪个键
	- 判断鼠标点击了哪个元素

**获取**
- 事件绑定函数的第一个参数就是**事件对象**
- 常命名为event, e

### 常见事件属性
- type
	- 当前事件类型
- clientX/clientY
	- 光标相对于**浏览器可见窗口**左上角位置
- offsetX/offsetY
	- 光标相对于**当前DOM元素**左上角位置
- key
	- 用户按下的**键盘的值**
	- 现在不提倡使用keyCode
	- key为string, keyCode是数值


# 环境对象
- 分析判断函数运行在不同环境中this代指的对象
- **环境对象**指的是**函数内部**特殊的**变量this**, 代表着当前环境运行时所处的环境
- this指向**函数的调用者**
- 函数的**默认this对象指向的是window对象**
	- 因为函数的默认调用者是window, 函数调用的完整写法为`window.fn();`

# 回调函数
- 如果将函数a**作为参数**传递给函数b, 称a为回调函数
